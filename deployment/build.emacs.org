Supplementary build notes for RHEL-compatible distributions.
For Ubuntu 24.04, use [[file:emacs-config-ubuntu24.sh]] instead.

Tested on: AlmaLinux 9, Rocky Linux 9, CentOS Stream 9.
CentOS 8 reached EOL December 2021 and is no longer supported.

* Prerequisites

** Enable required repositories

The CRB repository (CodeReady Builder, formerly "PowerTools" on CentOS 8)
provides development headers not in the base repos.
EPEL provides libtree-sitter-devel and libvterm-devel.

#+begin_src sh
  sudo dnf install epel-release
  sudo dnf config-manager --set-enabled crb
  sudo dnf update
#+end_src

** Install build dependencies

#+begin_src sh
  sudo dnf install \
    gcc gcc-c++ clang make autoconf automake texinfo \
    libgccjit libgccjit-devel \
    gtk3-devel \
    gnutls-devel \
    harfbuzz harfbuzz-devel \
    libXpm-devel \
    libjpeg-turbo-devel \
    libpng-devel \
    libtiff-devel \
    giflib-devel \
    librsvg2-devel \
    libxml2-devel \
    jansson jansson-devel \
    sqlite sqlite-devel \
    ncurses-devel \
    dbus-devel \
    gpm-devel \
    acl-devel \
    libtree-sitter-devel \
    libvterm-devel
#+end_src

Note: =libtree-sitter-devel= and =libvterm-devel= come from EPEL.
If EPEL does not yet carry them for your distro version, build from source:

#+begin_src sh
  # tree-sitter (if not available via EPEL)
  git clone --depth 1 --branch v0.24.4 https://github.com/tree-sitter/tree-sitter.git
  cd tree-sitter && make && sudo make install PREFIX=/usr/local
#+end_src

* Download sources

** Option A: Official tarball (stable, recommended)

#+begin_src sh
  EMACS_VERSION=30.2
  curl -LO "https://ftp.gnu.org/gnu/emacs/emacs-${EMACS_VERSION}.tar.xz"
  tar -xJf "emacs-${EMACS_VERSION}.tar.xz"
  cd "emacs-${EMACS_VERSION}"
#+end_src

** Option B: Git clone (any branch, tag, or commit)

#+begin_src sh
  # Stable release branch
  git clone --depth 1 --branch emacs-30.2 https://git.savannah.gnu.org/git/emacs.git
  # or: latest master
  git clone --depth 1 --branch master https://git.savannah.gnu.org/git/emacs.git
  cd emacs
  ./autogen.sh
#+end_src

* Build

#+begin_src sh
  INSTALL_PREFIX="${HOME}/.local"   # no sudo needed; adjust to taste

  CC=clang ./configure \
    --prefix="${INSTALL_PREFIX}" \
    --with-pgtk \
    --with-native-compilation=aot \
    --with-tree-sitter \
    --with-json \
    --with-modules \
    --with-sqlite3 \
    --with-harfbuzz \
    --with-compress-install \
    --with-threads \
    --with-rsvg \
    --with-jpeg \
    --with-png \
    --with-gif \
    --with-tiff \
    --with-xpm \
    --with-gnutls \
    --with-dbus \
    --with-mailutils

  make -j"$(nproc --ignore=2)"
  make install
#+end_src

Note: =--with-pgtk= requires GTK3 and enables Wayland support.
On systems without Wayland you can substitute =--with-x-toolkit=gtk3=.

* Packaging (optional)

Create a relocatable tarball for distribution to other machines:

#+begin_src sh
  INSTALL_STAGE="${HOME}/emacs-stage"
  make prefix="${INSTALL_STAGE}" install
  tar -cJf "emacs-30.2-rhel9.tar.xz" -C "${INSTALL_STAGE}" .
  echo "Deploy on another machine: tar -xJf emacs-30.2-rhel9.tar.xz -C ~/.local/"
#+end_src
