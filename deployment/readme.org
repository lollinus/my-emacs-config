* Overview

Three ways to get a current Emacs on Ubuntu (Linux focus). Choose based on
your needs:

| Path              | File                          | Effort   | Reproducible | Notes                        |
|-------------------+-------------------------------+----------+--------------+------------------------------|
| Snap              | [[file:snap-ubuntu.md]]          | Minimal  | Auto-updates | Recommended for daily use    |
| Script (stable)   | [[file:emacs-config-ubuntu24.sh]] | Medium   | Pinned ver.  | Emacs 30.2, installs ~/.local |
| Script (any ref)  | same script, =--ref REF=        | Medium   | No           | Branch, tag, or commit SHA   |
| Bazel (stable)    | [[file:bazel/BUILD.bazel]]        | High     | Yes (SHA256) | Cached, reproducible         |
| Bazel (any ref)   | same workspace, =EMACS_REF=     | High     | Per ref      | Cached by ref name           |

Target: Ubuntu 24.04 LTS, Emacs 30.x.

* Path 1: Snap (recommended for daily use)

The simplest path. Tracks the latest Emacs stable release automatically.

#+BEGIN_SRC sh
  sudo snap install emacs --classic
#+END_SRC

=--classic= is required for unrestricted filesystem access (GPG, tramp, vterm,
native compilation cache).

See [[file:snap-ubuntu.md]] for full details including first-launch notes and =local.el= setup.

* Path 2: Build from source — shell script

** Prerequisites

Install system packages once:

#+BEGIN_SRC sh
  sudo apt install $(grep -v '^#' emacs-pkgs-ubuntu24.txt | tr '\n' ' ')
#+END_SRC

See [[file:emacs-pkgs-ubuntu24.txt]] for the full annotated list.

** Build stable Emacs 30.2

#+BEGIN_SRC sh
  ./emacs-config-ubuntu24.sh
#+END_SRC

Clones or downloads Emacs 30.2, configures, builds with =make -j$(nproc --ignore=2)=,
and installs to =~/.local/=. Also produces a distributable =.tar.xz= and an uninstall script.

Configure highlights:
- =--with-pgtk= — Pure GTK; works on Wayland *and* X11
- =--with-native-compilation=aot= — ahead-of-time native compilation
- =--with-tree-sitter= — built-in tree-sitter support
- =--with-sqlite3= — needed by org-roam and consult backends

** Build latest git master (bleeding-edge)

#+BEGIN_SRC sh
  ./emacs-config-ubuntu24.sh --git
  # equivalent to: --ref master
#+END_SRC

** Build any branch, tag, or commit

#+BEGIN_SRC sh
  ./emacs-config-ubuntu24.sh --ref emacs-30       # release branch
  ./emacs-config-ubuntu24.sh --ref emacs-30.2     # release tag
  ./emacs-config-ubuntu24.sh --ref a1b2c3d4       # specific commit SHA
#+END_SRC

Branches and tags use a shallow clone (fast). Commit SHAs trigger a blobless
full clone since Savannah does not support direct SHA fetches.
Subsequent runs with the same ref update the existing clone in-place.

*Warning:* git-based builds may be unstable between releases.

** Add to PATH

#+BEGIN_SRC sh
  export PATH="${HOME}/.local/bin:${PATH}"
  # Add to ~/.bashrc or ~/.profile to persist
#+END_SRC

* Path 3: Bazel build

Bazel orchestrates fetching, building, and packaging Emacs. The stable build is
content-addressed and cached — subsequent =bazel build= calls are instant if
nothing has changed.

** Prerequisites

1. Install system packages (same as Path 2 above).

2. Install Bazel 7+ (recommend using Bazelisk):
   #+BEGIN_SRC sh
     npm install -g @bazel/bazelisk   # or: go install github.com/bazelbuild/bazelisk@latest
     # or download from https://github.com/bazelbuild/bazelisk/releases
   #+END_SRC

** Build stable Emacs 30.2

#+BEGIN_SRC sh
  cd deployment/bazel
  bazel build //:emacs_package
  # Output: bazel-bin/emacs-30.2-install.tar.xz
#+END_SRC

** Build from any git ref or commit

Use the =emacs_ref_package= target with the =EMACS_REF= environment variable.
Bazel caches each unique ref value separately (tracked via =--action_env=EMACS_REF=
in =.bazelrc=).

#+BEGIN_SRC sh
  cd deployment/bazel

  # git master (default when EMACS_REF is unset)
  EMACS_REF=master bazel build //:emacs_ref_package

  # release branch
  EMACS_REF=emacs-30 bazel build //:emacs_ref_package

  # release tag
  EMACS_REF=emacs-30.2 bazel build //:emacs_ref_package

  # specific commit SHA (triggers blobless full clone)
  EMACS_REF=a1b2c3d4e5f6 bazel build //:emacs_ref_package
#+END_SRC

Output is always =bazel-bin/emacs-ref-install.tar.xz=.

*Caching note:* mutable refs like =master= are cached by ref name, not by upstream
commit. To force a fresh build after upstream advances:

#+BEGIN_SRC sh
  EMACS_REF=master EMACS_REF_FORCE=$(date +%s) bazel build //:emacs_ref_package
#+END_SRC

** Deploy with the convenience script

#+BEGIN_SRC sh
  cd ~/.emacs.d/deployment/bazel

  ./deploy.sh                          # stable 30.2, clang (default)
  ./deploy.sh --cc gcc                 # build with GCC instead
  ./deploy.sh --ref master             # git master
  ./deploy.sh --ref emacs-30           # release branch
  ./deploy.sh --ref a1b2c3d4           # specific commit
  ./deploy.sh --ref master --force     # force rebuild (ignores cache)
  ./deploy.sh --ref emacs-30 --cc gcc  # ref + explicit compiler
#+END_SRC

The =--cc= flag selects the compiler (=clang= or =gcc=). Bazel caches clang and
gcc builds separately, so switching is instant if both have been built before.

Compiler can also be passed via environment variable:

#+BEGIN_SRC sh
  EMACS_CC=gcc bazel build //:emacs_package
#+END_SRC

* Fonts

All paths require the same fonts. Install once:

#+BEGIN_SRC sh
  sudo apt install unifont fonts-dejavu fonts-cantarell
  fc-cache -frv
#+END_SRC

To verify installed fonts:
#+BEGIN_SRC sh
  fc-list | grep -i "unifont\|dejavu\|cantarell"
#+END_SRC

** Manual font installation (without apt)

Place =.ttf= files in =~/.local/share/fonts/= then run =fc-cache -frv=.

- Unifont: https://unifoundry.com/unifont.html
- DejaVu: https://dejavu-fonts.github.io
- Cantarell: https://fonts.google.com/specimen/Cantarell

* Configuration deployment

After installing Emacs by any path above:

#+BEGIN_SRC sh
  # Clone this config (if not already present)
  git clone <repo-url> ~/.emacs.d

  # Copy the machine-local template and customise
  cp ~/.emacs.d/local.el.example ~/.emacs.d/local.el
  $EDITOR ~/.emacs.d/local.el
#+END_SRC

First launch will auto-install all MELPA/ELPA packages. This takes a few minutes.

* Uninstalling Emacs

** Bazel-built or script-built (installed to =~/.local/=)

Use =remove.sh= — the companion script to =deploy.sh= in the Bazel directory:

#+BEGIN_SRC sh
  cd ~/.emacs.d/deployment/bazel
  ./remove.sh           # remove Emacs 30.2 (default version)
  ./remove.sh 30.2      # explicit version argument
#+END_SRC

This removes:
- Binaries: =emacs=, =emacs-30.2=, =emacsclient=, =etags=, =ctags=, =ebrowse=
- Data directories: =~/.local/{share,lib,libexec}/emacs/30.2/=
- GLib schema: =org.gnu.emacs.defaults.gschema.xml= (recompiles the schema index)
- =~/.local/share/emacs/site-lisp/= only if it contains no user-added files

To also remove the Bazel build cache (the compiled artifact tarball):

#+BEGIN_SRC sh
  cd ~/.emacs.d/deployment/bazel
  bazel --nohome_rc clean            # removes bazel-bin/ outputs
  bazel --nohome_rc clean --expunge  # also removes the shared action cache
#+END_SRC

** Manual removal (one-liner)

#+BEGIN_SRC sh
  VERSION=30.2
  rm -f ~/.local/bin/emacs ~/.local/bin/emacs-${VERSION} \
        ~/.local/bin/emacsclient ~/.local/bin/etags \
        ~/.local/bin/ctags ~/.local/bin/ebrowse
  rm -rf ~/.local/share/emacs/${VERSION} \
         ~/.local/lib/emacs/${VERSION} \
         ~/.local/libexec/emacs/${VERSION}
  rm -f ~/.local/share/glib-2.0/schemas/org.gnu.emacs.defaults.gschema.xml
  glib-compile-schemas ~/.local/share/glib-2.0/schemas/ 2>/dev/null || true
#+END_SRC

** Snap installation

#+BEGIN_SRC sh
  sudo snap remove emacs
#+END_SRC
